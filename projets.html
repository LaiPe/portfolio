Tours de Hanoï

Site Web crée le 10/04/2025

Tours de Hanoï est un petit projet de site web gadget rattaché à ma formation de Concepteur Développeur d'Applications chez HumanBooster.

Le but de ce site est de permettre à l'utilisateur de visualiser le jeu des Tours de Hanoï.

Ce jeu de réflexion, inventé par le mathématicien Édouard Lucas, consiste à déplacer des disques de diamètres différents d'une tour de départ à une tour d'arrivée en suivant deux règles simples : on ne peut déplacer qu'un disque à la fois et on ne peut jamais placer un disque sur un autre plus petit que lui. Ce puzzle classique met en lumière des concepts fondamentaux d'algorithmique et de récursivité.

pálinDrómos

Site Web crée le 29/10/2024

pálinDrómos est un petit projet de site web gadget rattaché à la certification « JavaScript Algorithms and Data Structures » décernée par l'organisme freeCodeCamp.

Le but de ce site est de permettre à l'utilisateur de vérifier si un mot, une phrase, ou plus généralement une chaîne de caractère est un palindrome.

Également nommé "palindrome de lettres", c'est une figure de style désignant un mot ou une phrase dont l'ordre des lettres reste le même qu'on les lise de gauche à droite ou de droite à gauche, comme dans la phrase « Ésope reste ici et se repose » ou encore « La mariée ira mal » à un accent près.

Sim-rhinite

Simulation crée le 25/06/2024

Sim-rhinite est un projet de simulation de maladie infectieuse type rhinite. L’objectif principal de cette simulation est de modéliser la propagation d’un virus non mortel dans une population et de produire des statistiques à partir de ce modèle qui pourront être analysées ultérieurement.

Ce projet s'inscrit dans l'évaluation d'un enseignement dispensé en L2 Informatique par l'UCA nommé "Projet Informatique". Cet enseignement a pour sujet la réalisation de simulations scientifiques à l'aide d'outils informatiques.

Le projet est composé de deux programmes :

  • stats_sim : Il permet de produire des statistiques sur l'évolution de la contamination de cette maladie en fonction du nombre d'infectés initiaux.
  • idle : Annexe permettant de visualiser le comportement de la population simulée.

Le projet comprend également un fichier README contenant des informations relatives à la compilation et l'exécution des programmes du projet. Une rédaction complète est également disponible, présentant le problème, le développement des programmes du projet, les résultats d’exécution de plusieurs scénarios d'infections et leurs analyses.

Sokoban

Jeu Vidéo crée le 26/05/2023

Le jeu de Sokoban est un jeu de puzzle inventé au Japon dans les années 1980. Le joueur incarne un personnage qui doit pousser des caisses vers des emplacements spécifiques dans un entrepôt. L'objectif est de déplacer toutes les caisses sur les emplacements cibles pour terminer chaque niveau.

Réalisé dans le cadre de mes études, ce projet est une modélisation du jeu de Sokoban codée en C. Pour l'instant, le programme ne peut jouer qu'une partie de "test". Cependant, il n'est pas impossible que j'ajoute dans de futures mises à jour des modes de jeu plus élaborés.

De plus, le projet contient également une rédaction disponible au format PDF détaillant les choix d’implémentations et de conception du programme. À noter que depuis la rédaction de ce compte-rendu, le projet a connu des évolutions. Ainsi, certains points de la rédaction peuvent être désuets au regard de l'état actuel du projet. Cette rédaction est donc à replacer dans le contexte du projet tel qu'il était à sa version 0.1.0-alpha. Cette version est disponible ici ou bien en temps que release sur le dépot Github.

Le fichier README contient, en plus des règles du jeu, la liste des commandes ainsi que les instructions relatives à la compilation et à l'exécution du programme.

La Réussite des Alliances

Jeu Vidéo crée le 24/04/2022

La Réussite est un jeu de cartes solitaire. Dans ce projet, les règles de la Réussite ont été simplifiées mais restent dans les grandes lignes les mêmes : regrouper les cartes en piles et essayer de posséder le moins de piles possibles. Ainsi, à chaque tour de jeu, le joueur pioche une carte qu’il dispose sur un banc de cartes, les unes à la suite des autres. Une carte peut former une pile si les cartes adjacentes possèdent la même valeur ou la même enseigne (carreau, trèfle, cœur ou pique). Si une des deux conditions est remplie, la carte sélectionnée pourra alors se poser sur la carte qui la précède. Vous pourrez trouver de plus amples explications sur les règles du jeu dans la rédaction jointe au projet.

Ce projet est donc la modélisation informatique de cette version de la Réussite. Réalisé dans le cadre de mes études, il est codé en Python et est accompagné par un compte-rendu complet détaillant l’implémentation, les choix de conceptions, les options et autres fonctionnalités contenu dans le projet.

De plus, est joint à ce projet un fichier README donnant les instructions nécessaires au démarrage du programme.
